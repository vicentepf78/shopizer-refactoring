# ADR-004: Módulos thin sm-content-core / sm-merchant-core

## Status

Aceito

## Data

2026-07-18

## Contexto

A Onda 1 usou JARs thin core (ex.: `sm-reference-core`) entre `sm-core` e os serviços executáveis. A lógica de domínio de content e merchant hoje vive dentro de `sm-core` junto com módulos não relacionados. Extrair direto para apps Boot sem um módulo intermediário arrisca dependências Maven circulares e puxa beans desnecessários de catálogo/CMS de produto para merchant ou content.

STATE.md AD-011.

## Decisão

Criar **`sm-content-core`** e **`sm-merchant-core`** como módulos Maven thin que extraem o subset relevante de `sm-core` (repositórios, serviços, backends CMS de conteúdo para content; serviços merchant/configuration/log para merchant). Os executáveis `content-service` e `merchant-service` dependem desses cores. `search-service` **não** depende de nenhum dos dois. `sm-core` mantém dependência de compile nos thin cores após a movimentação para callers in-process restantes até o cutover strangler completar.

## Alternativas consideradas

### Alternativa 1: Copiar serviços direto para apps Boot (sem thin core)

- **Descrição**: Mover classes apenas para `content-service` / `merchant-service`.
- **Prós**: Menos módulos.
- **Contras**: Reúso mais difícil a partir de sm-core durante dual-run; diverge do padrão da Onda 1.
- **Por que rejeitada**: O padrão thin-core da Onda 1 já está comprovado.

### Alternativa 2: Um `sm-wave2-core` compartilhado

- **Descrição**: Um único JAR para content + merchant.
- **Prós**: Um módulo para versionar.
- **Contras**: Acopla domínios não relacionados; bloqueia evolução independente.
- **Por que rejeitada**: Viola os limites de serviço.

## Consequências

### Positivas

- Evita deps circulares; search permanece livre de sm-core.
- Permite o split do CMS XML (`shopizer-content-cms.xml` vs remanescente só de produto).

### Negativas

- Módulos extras no POM do reactor.

### Riscos

- Movimentação incompleta deixa classes duplicadas — mitigar com gates de módulo (`./mvnw test -pl sm-*-core`).

## Notas de implementação

- Content core exclui `productFileManager` / downloads de produto.
- Merchant core remove a injection morta de `ProductTypeService` de `MerchantStoreServiceImpl`.
- IDs de requisito: CNT-05, MCH-01, STR-03.
